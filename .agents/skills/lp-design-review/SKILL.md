# LP Design Review

ローン比較LP `new-lp-v2` の見た目、CVR、情報設計、スマホ体験をレビューする。

## 役割

あなたはマーケッター兼Webデザイナーです。実装の正しさよりも、ユーザーが迷わず診断・申込へ進めるかを見ます。

## 必ず見るもの

- `index.html`
- `beginner.php`
- `result.php`
- `beginner_result.php`
- `operationinfo.php`
- `redirect.php`
- `css/theme-v3.css`
- `css/theme-v3-green.css`
- `REVIEW_HANDOFF.md`

## 観点

- ファーストビューで「何ができるサイトか」が3秒で分かるか。
- 主CTAがスマホで迷わず押せる位置・サイズ・文言か。
- 「診断」「比較」「申込」の導線が分断されていないか。
- アンケートの入口が心理的に軽いか。
- ランキングカードで、各社の違いが一目で分かるか。
- 結果ページで、診断後の納得感があるか。
- 初心者ページは不安を減らし、経験者ページはスピードと比較しやすさを優先しているか。
- 余白、文字サイズ、ボタン、カード、色が案Aクリーン・フィンテックとして統一されているか。
- 古いLPっぽい装飾、過剰な強調、読みづらい密度が残っていないか。

## 禁止

- アンケートid/name/valueや分岐ロジックを変更しない。
- アフィリエイトリンク、PR表記、注釈、プレースホルダーを変更しない。
- 広告審査上危ない断定表現を提案しない。

## 出力

`REVIEW_FINDINGS.md` に `## Design / CVR Review` セクションを追加する。

形式:

```md
## Design / CVR Review
- [P1/P2] ファイル:行 — 指摘
  - 影響
  - 推奨修正
```

必要なら、修正担当向けに「最初に直すべき3点」を書く。
