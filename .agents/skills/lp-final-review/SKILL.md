# LP Final Review

ローン比較LP `new-lp-v2` の実装後レビュー専用Skill。実装者とは別視点で、バグ・表示崩れ・文言・導線・禁止事項違反を検出する。

## レビュー姿勢

- まず修正しない。指摘を出す。
- Findings first。重大度順に、ファイルと行番号を添えて書く。
- ユーザーが明示的に修正まで依頼した場合だけ、指摘後に最小差分で修正する。
- 好みの話だけで終わらせず、CVR、スマホ可読性、法務/広告表記、診断ロジック、アフィリエイト導線のリスクに結びつけて判断する。

## 必ず読むもの

1. `AGENTS.md`
2. `CODEX_PROMPT.md`
3. `CODEX_TASKS.md`
4. `REVIEW_HANDOFF.md`
5. `git diff` / `git status` / 直近コミットログ
6. 主要ページと新規CSS/JS

Gitが dubious ownership で止まる場合は、その事実を `REVIEW_FINDINGS.md` に記録する。ユーザー承認なしにグローバル設定を変更しない。

## 重点確認

### P0: 壊れると致命的

- アンケートの `q-amount`〜`q-duration` が維持されているか。
- 指定 `name` が維持されているか。
- `ul.survey-list > li` の順序と段階表示が壊れていないか。
- `.entry-modal`, `.entry-first`, `.entry-experienced`, `#survey-modal` が維持されているか。
- `.loan-check input[name="current_loans[]"]` のvalueが変わっていないか。
- `redirect.php` の `linkMap`, `getStoredQueryParams`, `setTimeout`, `fallback-link` の転送ロジックが壊れていないか。
- アフィリエイトバナー、口コミ画像、PR表記、注釈、プレースホルダーが消えていないか。

### P1: CVRや信頼性に強く影響

- ヒーローで何のサイトか、何ができるか、次に何を押すかがスマホファーストビューで分かるか。
- 診断CTAが迷わず押せるか。
- アンケートの未回答状態、回答済み状態、進捗が分かりやすいか。
- 結果ページで「なぜこの3社か」「どこを押すか」が明確か。
- `operationinfo.php` が信頼情報として自然に読めるか。
- `redirect.php` が不安を与えない中継画面になっているか。

### P2: 品質改善

- 320 / 375 / 390 / 414pxで横スクロールがないか。
- sticky header、safe-area、追従CTA、モーダルが重ならないか。
- ボタンのタップ領域が44px以上あるか。
- テキストがボタンやカード内で詰まっていないか。
- Noto Sans JP、色、カード、余白、角丸が案Aクリーン・フィンテックに統一されているか。
- 青テーマと緑テーマの差分が意図通りか。

### 文言校正

`REVIEW_HANDOFF.md` の「新規に書いた文言」を中心に、以下を確認する。

- 日本語として自然か。
- 金融広告として断定しすぎていないか。
- 「必ず」「誰でも」「審査なし」など不適切な表現がないか。
- 「3問」「30秒」などの表現が実際の設問数・体験と矛盾していないか。
- 初心者向けと経験者向けのトーンが合っているか。

## 出力

レビュー結果は `REVIEW_FINDINGS.md` に保存する。チャットにも要約を返す。

`REVIEW_FINDINGS.md` の形式:

```md
# REVIEW_FINDINGS

## 結論
- 合格 / 条件付き合格 / 要修正 のいずれか

## Findings
- [P0/P1/P2] ファイル:行 — 問題
  - 影響
  - 推奨修正

## 文言校正
- 設置場所: 現文言 → 推奨文言

## 動作確認
- 実施した確認
- 未実施の確認

## 修正担当への指示
- 次に直すべき順番
```

重大な問題がなければ、明確に「重大な問題は見つかりません」と書き、残るリスクだけを短く記録する。
